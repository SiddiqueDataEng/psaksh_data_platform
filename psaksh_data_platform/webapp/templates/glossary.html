📖 Glossary & Data Dictionary

Complete reference for all indicators, methods, data engineering terms, and KPIs used in this platform

🥗 Child Nutrition Indicators

TermDefinitionThresholdSource
Stunting
HAZ < -2 SD		 Low height-for-age. Reflects chronic malnutrition — cumulative effect of inadequate nutrition over months/years. Largely irreversible after age 2. Indicates long-term deprivation. HAZ < -2 SD
Severe: HAZ < -3 SD		 WHO Child Growth Standards
Wasting
WHZ < -2 SD		 Low weight-for-height. Reflects acute malnutrition — recent food insecurity or illness. Life-threatening if severe. Responds quickly to therapeutic feeding. WHZ < -2 SD
Severe: WHZ < -3 SD		 WHO Child Growth Standards
Underweight
WAZ < -2 SD		 Low weight-for-age. Composite indicator capturing both stunting and wasting. Less specific than HAZ or WHZ but widely used for programme monitoring. WAZ < -2 SD WHO Child Growth Standards
Z-Score (HAZ/WAZ/WHZ) Number of standard deviations from the WHO median reference population. HAZ = Height-for-Age Z-score. WAZ = Weight-for-Age. WHZ = Weight-for-Height. Values beyond ±6 are flagged as implausible and nulled during ETL. Plausible range: -6 to +6 WHO 2006 Growth Standards
MUAC
Mid-Upper Arm Circumference		 Rapid screening tool for acute malnutrition. Measured in mm. <115mm = severe acute malnutrition. 115-125mm = moderate. Used in community-based screening. Historical data stored in cm — ETL converts to mm. SAM: <115mm
MAM: 115-125mm		 WHO/UNICEF
Anemia Hemoglobin below threshold. Child anemia: Hb <11 g/dL. Maternal anemia: Hb <11 g/dL. Causes: iron deficiency, malaria, infections. Increases risk of poor cognitive development and maternal mortality. Child: Hb <11 g/dL
Maternal: Hb <11 g/dL		 WHO 2011
Vaccination (Full) Child has received all recommended vaccines per national immunization schedule (BCG, OPV, DPT, measles). Binary indicator: 1=fully vaccinated, 0=not fully vaccinated. Target: ≥90% coverage EPI Pakistan
Exclusive Breastfeeding (EBF) Infant under 6 months receiving only breast milk (no water, formula, or other foods). Protects against diarrhea, respiratory infections, and malnutrition. Only recorded for children <6 months. Target: ≥50% under 6m WHO/UNICEF

🤱 Maternal Health Indicators

TermDefinitionTargetSource
ANC 4+
Antenatal Care ≥4 visits		 Proportion of women who received at least 4 antenatal care visits during pregnancy. WHO recommends ≥8 contacts. ANC 4+ is the standard programme indicator. Linked to lower maternal and neonatal mortality. WHO target: ≥80% WHO ANC Guidelines 2016
Skilled Delivery
Skilled Birth Attendant		 Delivery attended by a skilled health professional (doctor, nurse, midwife). Reduces maternal mortality by enabling management of obstetric complications. Key SDG 3.1 indicator. National target: ≥80% SDG 3.1 / PDHS
Hemoglobin (g/dL) Blood hemoglobin concentration in grams per deciliter. Used to diagnose anemia. Measured at ANC visits. Values <3 or >20 g/dL are flagged as implausible during ETL Silver cleaning. Normal: ≥11 g/dL WHO 2011

⚙ Data Engineering & ETL Terms

TermDefinitionUsed In
Medallion Architecture Three-layer data architecture: Bronze (raw ingestion, exact copy), Silver (cleaned, validated, schema-unified), Gold (analytics-ready facts, dimensions, KPIs). Each layer adds value without destroying the previous. ETL Pipeline
Bronze Layer 🥉 Exact copy of all source data with metadata added (_source_file, _ingested_at, _source_era, _load_type). No transformation. Preserves original errors for audit. Sources: CSV (SurveyCTO), Parquet (DB exports), JSON (DHIS2), legacy files. data/bronze/
Silver Layer 🥈 Cleaned, validated, schema-unified data. All DQ issues fixed: bilingual normalisation, date parsing, deduplication, outlier removal, GPS validation. CDC columns added. Saved as Parquet. data/silver/
Gold Layer 🥇 Analytics-ready data: fact tables (fct_child_nutrition, fct_maternal_health), SCD2 dimensions (dim_district, dim_facility), windowed KPI aggregations (rpt_district_summary, rpt_province_summary). Served via API to Power BI, Tableau, Streamlit. data/gold/
Full Load Load ALL data from source regardless of what was previously loaded. Used for initial setup or when source data has changed completely. Slower but ensures completeness. Triggered when no watermark exists. Bronze ingestion
Incremental Load Load only NEW records since the last successful run (tracked by watermark timestamp). Faster and more efficient than full load. Detects changes via file checksum comparison. Falls back to full load if checksum differs significantly. Bronze ingestion
CDC (Change Data Capture) Tracks every change to data: INSERT (new record), UPDATE (modified record), DELETE (removed record). Each Silver record gets _cdc_op, _cdc_ts, _cdc_source columns. Enables point-in-time recovery and audit trails. Silver layer
SCD Type 2
Slowly Changing Dimension		 Dimension table that preserves full history of changes. When a district or facility attribute changes, the old row is expired (_scd_effective_to set, _scd_is_current=False) and a new row is inserted. Enables historical analysis. dim_district, dim_facility
Delta Lake Pattern Data stored in partitioned directories (year=YYYY/month=MM/) with an audit log (pipeline_state.json) tracking every run. Enables time-travel queries, incremental processing, and ACID-like guarantees without a full Delta Lake engine. data/delta_log/
Watermark Timestamp of the last successful data load for each dataset. Stored in pipeline_state.json. Used to determine which records are "new" in incremental loads. Updated after each successful Bronze ingestion. pipeline_state.json
Upsert / Merge INSERT new records + UPDATE existing records in a single operation. Used in Gold layer to merge incoming data with existing fact tables. MySQL uses INSERT ... ON DUPLICATE KEY UPDATE. SQLite uses replace strategy. Gold layer, DB load
Schema Evolution Handling different column names and structures across data sources and years. The Silver layer applies a SCHEMA_MAP to rename legacy columns (e.g. hh_id→household_id, anaemia→anemia, muac_cm→muac_mm) to canonical names. Silver transform
Windowing / Time-Window Aggregation Aggregating data over defined time periods: monthly (rolling 30-day), quarterly (Q1-Q4), annual (year-over-year). Used in Gold layer to build rpt_district_summary (by visit_round) and rpt_annual_trend (by survey_year). Gold KPI tables
Data Lineage Every record tracks its origin: _source_file (original filename), _source_era (historical/current), _source_type (legacy_heterogeneous/mysql_db_export), _ingested_at (timestamp), _load_type (full/incremental/cdc). Enables full audit trail. All layers
Parquet Format Columnar storage format optimised for analytics. Compressed (typically 5-10x smaller than CSV). Preserves data types. Used for Bronze, Silver, and Gold layers. Enables fast column-level queries by Power BI, Pandas, Spark. Bronze/Silver/Gold

🔍 Data Quality Terms

TermDefinitionHow Fixed
Bilingual Field Values Field values entered in Urdu script instead of English (e.g. "پائپ" instead of "piped", "ہاں" instead of "1"). Common in Pakistan field surveys where enumerators switch languages. ~15% of records affected. Silver: URDU_TO_ENG maps
Duplicate Submissions Same household submitted multiple times (paper form entered twice, network retry, enumerator error). ~4.5% of records. Silver layer deduplicates on household_id keeping the latest submission_time. Silver: drop_duplicates(keep=last)
Anthropometric Outliers Biologically implausible measurements: height <40cm or >130cm for children, weight <1.5kg or >30kg, MUAC <80mm or >200mm, Z-scores |z|>6. Nulled during Silver cleaning with audit log. Silver: clip to plausible range
GPS Out-of-Bounds GPS coordinates outside Pakistan bounding box (lat 23.5-37.5°N, lon 60.5-77.5°E). Caused by GPS signal loss, manual entry errors, or test data. Nulled during Silver cleaning. Silver: null if outside bounds
Short Interview Flag Interview duration <10 minutes flagged as potential data fabrication. Enumerators may rush or fabricate responses. Preserved as short_interview_flag=1 in Silver/Gold for supervisor review. ~5% of records. Preserved as quality flag
Inconsistent Date Formats Historical data used many date formats: DD/MM/YYYY, MM-DD-YYYY, DD.MM.YYYY, YYYY/MM/DD. Silver layer tries 7 date formats in order, falling back to pandas auto-parse. ~8% of historical records affected. Silver: multi-format parser

🗺 Geography & Socioeconomic Terms

TermDefinition
SES Tier
Socioeconomic Status		 Household wealth classification: Low (58% of households — poorest quintiles, limited assets), Middle (32% — moderate assets, some formal employment), High (10% — formal employment, durable assets). Based on asset index, water source, sanitation.
Union Council (UC) Smallest administrative unit in Pakistan. Each district has multiple UCs. 5 UCs per district in this programme = 180 UCs total across 36 districts. Used for geographic targeting and coverage tracking.
DHQ / RHC / BHU DHQ = District Headquarters Hospital (highest level, 15 staff required). RHC = Rural Health Centre (secondary level, 7 staff). BHU = Basic Health Unit (primary level, 3 staff). Readiness scores: DHQ ~72%, RHC ~58%, BHU ~42%.
Provinces Pakistan's 4 provinces with distinct health profiles: Punjab (53% population, best indicators), Sindh (23%, urban-rural divide), KPK (15%, improving), Balochistan (9%, worst indicators — stunting 52%, skilled delivery 45%).

📡 API & BI Tool Integration

TermDefinitionEndpoint
Power BI Live Connection Power BI connects to the REST API via "Get Data → Web". The API returns OData-compatible JSON. Use the district-summary endpoint for pre-aggregated KPIs, or child-nutrition for row-level data. Refresh on schedule via Power BI Service. /api/v1/district-summary
Tableau WDC Tableau Web Data Connector endpoint returns data with column type hints (int/float/string/datetime). Use "Connect → Web Data Connector" in Tableau Desktop. Supports filtering via URL parameters. /api/v1/tableau/child-nutrition
OData-Compatible Filtering All API endpoints support URL parameter filtering: ?district=Lahore&visit_round=1. Pagination: ?limit=1000&offset=0. Field selection: ?fields=district,stunting_rate. Authentication: ?api_key=demo. All /api/v1/* endpoints
Streamlit Dashboard Python-based interactive dashboard reading directly from Gold Parquet files. Provides deeper analytical capabilities than Flask: interactive filters, cross-filtering, statistical analysis. Run: streamlit run dashboards/app.py Port 8501
